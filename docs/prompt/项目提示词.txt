我要做一个"江南水乡"2D 在线游戏的智能导游系统。

游戏背景：
- 玩家进入游戏后，出现在江南水乡场景（小桥流水、乌篷船、青石板路）
- 一个穿着古风裙的 NPC 少女导游站在桥头迎接
- 导游可以：欢迎玩家、回答游戏基础问题、引导玩家操作

技术栈：
- 后端：Go 1.22 + Gin + Gorilla WebSocket
- 前端：Phaser 3 (HTML5 Canvas) + 原生 JS
- 数据库：MySQL
- LLM：glm-4.7

导游 Agent 能力：
1. 欢迎接待：识别玩家首次进入，主动欢迎，介绍游戏背景
2. 基础问答：游戏怎么玩、怎么移动、有什么任务、怎么赚钱等
3. 上下文记忆：记住玩家昵称、上次对话内容
4. 情绪感知：根据玩家输入判断情绪（开心/困惑/生气），调整回复语气

简化要求：
- 不要 RAG，知识库用硬编码 JSON
- 不要复杂工作流，单轮/多轮对话即可
- 保留审计日志（记录每次对话，用于分析）
- OpenTelemetry 成本追踪
- 熔断降级： LLM 服务异常时，切换备用模型或返回兜底回复                 
- 超时控制：每层都有独立超时：总请求 30s → LLM 调用 10s → 工具调用 5s 
- 成本控制：相似问题缓存回答（Embedding 相似度 > 0.95） 、上下文压缩：摘要历史消息，只保留关键信息         
- 资源隔离：不同租户独立线程池，防止资源争抢                       
- 代码要简单清晰，适合学习
