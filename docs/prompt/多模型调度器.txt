
## 核心要求

### 1. model 层（必须）
- 定义 `Provider` 接口：`Name(), AvailableModels(), Chat(), StreamChat(), InputPricePer1K(), OutputPricePer1K(), MaxContextLength()`
- 统一 `ChatRequest/ChatResponse`，支持 Tools/Function Calling
- `StreamChunk` 支持 SSE 流式输出

### 2. providers 层（至少 Claude + OpenAI）
- Claude 实现用 `github.com/anthropics/anthropic-sdk-go`
- OpenAI 实现用 `github.com/sashabaranov/go-openai`
- 都要支持：普通 Chat、流式 Chat、Tool Calling
- 价格数据可配置

### 3. router 层（核心亮点）
- 支持策略：Fixed / Cost / Latency / Capability / Fallback / Weighted
- `ModelStats` 用指数移动平均跟踪延迟和错误率
- `RouteRequest()` 根据策略返回 `(ChatRequest, Provider, error)`
- 降级链：主模型 -> 通用备选 -> 低成本兜底
- 任务分类函数：根据关键词识别 code/reasoning/chinese/longtext

### 4. server 层（必须）
- 标准 HTTP 服务，端口 8080
- `/chat` 接口：JSON 请求，JSON 响应
- `/chat/stream` 接口：SSE 流式响应
- 中间件：请求日志（带 trace_id）、panic 恢复
- 支持通过 header `X-Route-Strategy` 选择策略

### 5. agent 层（加分项）
- 简单 ReAct 状态机：Thought -> Action -> Observation -> Answer
- 支持 2-3 个工具：计算器（本地）、天气查询（模拟）、搜索引擎（模拟）
- 记忆管理：滑动窗口 + 粗略 Token 计数截断

### 6. rag 层（加分项）
- 简单向量检索：用内存 map 模拟（面试时不用接真实向量库）
- HybridSearch 框架：向量 + BM25 + RRF 融合（BM25 和 RRF 可以简化实现）
- 文档分块：固定长度 + 重叠

## 代码风格要求
- 所有导出类型和方法必须有文档注释
- 错误处理完整，不要忽略 err
- 并发安全：router stats 用 sync.RWMutex
- 不要引入重型框架（不用 gin、不用 langchaingo）
- 标准库为主：`net/http`, `encoding/json`, `sync`, `context`
- 第三方库只限：anthropic-sdk-go, go-openai

## 面试亮点注释
在关键代码处加注释，说明设计意图，例如：
- `// 指数移动平均：新样本权重 30%，历史权重 70%，平滑异常波动`
- `// 降级链容忍更高错误率，确保可用性优先于成本`
- `// Token 估算：每4字符约1token，中文按字节估算`
